# Blog Mobile Sitemaps

Auto-generated mobile sitemap (`?m=1`) hosting for Finsaathi Editorial blogs.

## Why m=1 Sitemap?

Blogger automatically redirects mobile UA (including Googlebot Smartphone, the mobile-first indexer) from clean URLs to `?m=1` versions. This creates a canonical-redirect loop when:
- Sitemap submits clean URLs to GSC
- Server redirects mobile crawlers to `?m=1`
- m=1 page canonical points back to clean URL
- Loop → "Redirect error" or "URL is unknown to Google"

Submitting `?m=1` URLs directly bypasses this loop.

## Sitemaps

- **PaisaPath**: [paisapath/sitemap-mobile.xml](paisapath/sitemap-mobile.xml)
- **Rupee Central**: [rupeecentral/sitemap-mobile.xml](rupeecentral/sitemap-mobile.xml)

## Auto-Update

These sitemaps are auto-regenerated by the `blogger-publish` skill after each new post is published. The skill calls Blogger API to fetch the current post list and commits an updated sitemap here.

See: `jin.kang/skills/blogger-publish/`

## GSC Registration

Both sitemap URLs are registered as cross-domain sitemaps in Google Search Console:
- PP property: `https://blog-sitemaps.github.io/paisapath/sitemap-mobile.xml`
- RC property: `https://blog-sitemaps.github.io/rupeecentral/sitemap-mobile.xml`
